## рҹ§  Wie funktioniert NETSHIELD?

NETSHIELD aggregiert **98 Г¶ffentliche Threat-Intelligence-Feeds**, bewertet jede IP nach einem mehrdimensionalen Scoring-Modell und erzeugt daraus qualitativ hochwertige, automatisch berechnete Blocklisten.

### Das Kernprinzip

```
Nur HQ-Feeds bestimmen die Lebenszeit einer IP.
Statische Mega-Listen erhГ¶hen den Score вҖ” aber sie kГ¶nnen eine IP nicht am Leben halten.
Das System bereinigt automatisch, was die Feeds selbst nicht kГ¶nnen.
```

IPs ohne HQ-BestГӨtigung altern nach **180 Tagen** automatisch aus. Werden sie erneut von einem HQ-Feed gemeldet, kehren sie sofort zurГјck.

## рҹ“Ҡ Confidence-Score

Jede IP erhГӨlt einen **Score von 0вҖ“100** aus vier Dimensionen:

```
Score = A (Quellen-QualitГӨt) + B (AktualitГӨt) + C (Persistenz) + D (Bekannt seit)
```

| Dimension | Max | Beschreibung |
|:---|:---:|:---|
| **A вҖ” Quellen-QualitГӨt** | 40 | HQ-Feed = 40 Pkt В· mehrere Feeds = 20вҖ“35 Pkt |
| **B вҖ” AktualitГӨt** | 30 | HQ-BestГӨtigung heute = 30 В· vor 7 Tagen = 20 В· vor 30 Tagen = 6 |
| **C вҖ” Persistenz** | 20 | BestГӨtigt Гјber 14+ Tage = 20 Pkt |
| **D вҖ” Bekannt seit** | 10 | LГӨnger im System = stabilerer Score |

### Score-Schwellwerte & Verwendung

| Score | Liste | Einsatz |
|:---:|:---|:---|
| **вүҘ 65** | `active_blacklist` | рҹ”ҙ OPNsense В· pfSense В· iptables |
| **вүҘ 40** | `blacklist_confidence40` | рҹҹЎ Erweiterte Filter В· IDS/IPS |
| **25вҖ“39** | `watchlist` | рҹҹ  Monitoring В· SIEM |
